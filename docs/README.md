**現行版: v3.2（2026-05-21 改訂）** — v3.1 への三次レビュー（CTO・セキュリティ・法務・App Store 審査担当の合議形式）指摘を反映。「もしもの時」発動フロー（emergencyMode）は引き続き Feature Flag で β段階停止し、本人主導の手動共有のみを提供。v3.2 の主な変更点:

1. **「将来提供予定」文言を「仮に提供する場合は…」へ書き換え（債務発生語の除去）** — ToS 第3条第5項・第7条第6項・第7項、PP 第4条第4号・第18条第7項。将来機能を提供することを運営者が約束するものではない旨を明示し、景品表示法・消費者契約法第8条の2 上のリスクを最小化
2. **OTP の法的位置付けを「メールアドレス到達確認」に統一** — ToS 第6条第5項第3号で「身元（本人性）を保証する本人確認手続ではない」と明示。invitation-flow-design.md §7.5 の表に法的位置付けの注記を追加
3. **家族ラベル選択時のUI警告を強化** — invitation-flow-design.md §4.1.2 招待発行モーダルの注記を「**法定相続人の指定や代理権の付与・本人の代わりに契約を解約する権限の付与を意味するものではありません**」へ書き換え
4. **β公開のハードゲート3点を README に明示** — データエクスポート・アカウント削除導線・同意取得画面はいずれも規約で約束済みのため、未実装のままβ公開すると債務不履行リスク
5. **Phase 4 課金開始前の事前チェックリストを新設** — 特商法・バーチャルオフィス・返金ポリシー・サブスク解約導線・IAP 等の課金前タスクを README に追記
6. **App Store / Google Play 申請前必須対応リストを legal-review-checklist 第5セクションに追加** — Privacy Manifest（2024/5 必須）、5.1.1(v) アカウント削除導線、マーケ文言審査対策、ATT、データセーフティ申告

### 🚨 β公開のハードゲート（規約で約束済み・未実装＝債務不履行リスク）

以下3点は v3.x 規約・PP で利用者に提供を約束している機能である。**未実装のままβ公開すると、規約と実装の乖離により消費者契約法第10条に基づく無効主張・債務不履行責任のリスクが発生する**。β公開前に必ず実装完了させること。実装が間に合わない場合は、heartbeat 機構と同様に規約を改定し「将来導入予定」に格下げすること。

- [ ] **【最優先】データエクスポート機能（JSON形式・本人のみ実行）の β実装完了**（ToS 第15条第7項・PP 第14条第5項で提供を明文化済み）
- [ ] **【最優先】アカウント内アカウント削除導線の β実装完了**（PP 第13条第1項・第13条第2項で提供を明文化済み。App Store 5.1.1(v) / Google Play との二重要件）
- [ ] **【最優先】同意取得画面（チェックボックス2分離・事前チェックなし・Dark Pattern 回避）の β実装完了**（PP 第18条 各号で要件明文化済み）

### Phase 4（有料化）開始前の事前チェックリスト

無料β・無料一般PWA段階では不要。Phase 4 で月額課金等を開始する **直前** に以下を整備すること。これらが整わない限り課金を開始してはならない。

- [ ] 特商法に基づく表記ページ作成（販売価格・支払方法・支払時期・引渡時期・返品特約・運営者名・住所・電話番号・代表者名）
- [ ] バーチャルオフィス契約（個人住所を公衆に開示するリスク回避のため。月額3,000円程度）
- [ ] 返金・キャンセルポリシー策定（サブスクの月途中解約時の取扱い・日割り返金の有無を明示）
- [ ] サブスク解約導線の実装（**アプリ内2タップ以内**で解約に到達。改正特商法・App Store 3.1.2 と整合）
- [ ] 決済はプラットフォーム強制（Apple IAP / Google Play Billing）。Stripe 等の外部決済はストア配信下では不可
- [ ] 適格請求書発行事業者登録の要否判断（B2C 個人なら登録不要。B2B 想定または年商1,000万円超で要検討）
- [ ] 消費者法専門弁護士による特商法表示・サブスク解約導線レビュー
